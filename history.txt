requirements.txt is installed by:
pip install -r requirements.txt

and check:
pip freeze

project superlists created by:
django-admin.py startproject superlists .

the book says:
Functional Test == Acceptance Test == End-to-End Test

workflow:
1) write FunctionalT (what user wants) -> fails
2) write UnitT (what the code should be doing) -> fail
3) write code that makes the UT pass - repeat 2&3 so FT can pass
4) run FT - passes
